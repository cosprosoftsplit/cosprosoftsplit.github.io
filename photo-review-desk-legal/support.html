Photo Review Desk Support

Common questions

Why does Photo Review Desk ask for Photo Library access?

Photo Review Desk reads asset metadata (size, creation date, media type) from the photos and videos you allow. The first time you tap Start Review, iOS asks how much access to give. Limited library access is supported as a first-class state — you can review only the items you choose.

Will Photo Review Desk delete anything by itself?

No. Every deletion is confirmed twice: first by you in Photo Review Desk, then by iOS in the system Photos deletion sheet. Photo Review Desk uses Apple's PHAssetChangeRequest.deleteAssets API which always shows the system dialog.

What if I have only a few items shared via Limited Photos access?

That works. Photo Review Desk enumerates only the items you shared. The review queues will show only those items.

Will Photo Review Desk make my phone faster or free system memory?

No. iOS does not allow third-party apps to free system memory, clear other apps' caches, scan for malware, or remove system files. Photo Review Desk is a Photos review utility only.

How is similar photo grouping done?

Photo Review Desk uses Apple's on-device Vision feature print API. Similarity is local. Items are labeled as possible duplicates because automatic grouping is not perfect — the final keep / delete decision is always yours.

How is the blur queue built?

Photo Review Desk applies a conservative local heuristic via Apple's CoreImage APIs. Items are labeled as possible blur. The final keep / delete decision is always yours.

How do I clear local cleanup records?

Settings → Clear analysis cache removes every local cleanup record, summary, and thumbnail reference from this device. Clearing the analysis cache does not delete any item from your Photos library.
